Detailed Description

Represents occupation probability via log-odds. This implements similar logic to what the OctoMap library uses.

Note
Supports float and double data types only.

Parameters
propertiesShared, constant pointer to the Grid::Properties to use.
dist_minMinimum update distance. Default -0.2.
dist_maxMaximum update distance. Default +0.2.
p_maxProbability for voxel maximum voxel value saturation. Default 0.98.
p_minProbability for voxel minimum voxel value saturation. Default 0.02.
p_pastProbability for voxels pst the sensed point. Default 0.50.
p_sensedProbability for the voxel at the sensed point. Default 0.90.
p_farProbability for voxels far in front of the sensed point. Default 0.15.
p_initProbability for voxel initialization. Default 0.50.
save_as_log_oddsIf true then the intermediate log odds value is stored. Other wise the voxels are converted to probability, saved, then back to log odds.
type_idDatatype for the Grid. Default is float.
